# Code of Conduct

## The short version

Be kind. Be constructive. Skills can be irreverent (see: gordon-ramsay, conspiracy) but contributors can't.

## The longer version

This project follows the [Contributor Covenant](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) v2.1. The key points:

- **Be respectful.** Disagreements about code are fine. Personal attacks are not.
- **Be constructive.** "This skill is too vague" with specifics is helpful. "This is bad" is not.
- **No harassment.** Of any kind, for any reason.
- **Skills vs. people.** Skills like `gordon-ramsay` and `passive-aggressive` insult *code*, never developers. The same standard applies to contributors: critique the work, not the person.

## Enforcement

Violations can be reported by opening an issue or contacting the maintainers directly. We'll address them.

## Scope

This applies to all project spaces: issues, PRs, discussions, and any public context where you're representing this project.
